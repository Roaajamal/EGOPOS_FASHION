## 2. ما يجب فعله لتشغيل التطبيق

### الخطوة 1: إنشاء ملف `.env`
- تم إنشاء ملف **`.env.example`** في المشروع (يحتوي على قيم مناسبة لبيئتك).
- انسخه إلى `.env`:
  ```bash
  copy .env.example .env
  ```
- أو انسخ المحتوى يدوياً إلى ملف جديد باسم `.env`.

### الخطوة 2: تثبيت الحزم
```bash
cd c:\xampp\htdocs\ego-pos
composer install
```

### الخطوة 3: توليد مفتاح التطبيق
```bash
php artisan key:generate
```

### الخطوة 4: إنشاء مجلدات التخزين (إن لم تُنشأ تلقائياً)
```bash
mkdir storage\framework\sessions
mkdir storage\framework\views
mkdir storage\framework\cache
mkdir storage\logs
mkdir storage\app\public
```
أو قد تنشئها أوامر Laravel تلقائياً عند أول تشغيل.

### الخطوة 5: التأكد من إعدادات `.env`
تأكد أن في `.env`:
- `APP_URL=http://localhost:8521` (أو الرابط الذي تستخدمه للبورت 8521)
- `DB_DATABASE=testv02`
- `DB_USERNAME` و `DB_PASSWORD` حسب إعداد MySQL عندك

### الخطوة 6: تشغيل الهجرات
```bash
php artisan migrate
```

### الخطوة 7: تشغيل التطبيق
- إما من Apache على البورت 8521 (إعداد Virtual Host كما في تحليل_ego-pos.md).
- أو:
  ```bash
  php artisan serve --port=8521
  ```

## 3. ملخص

- **التطبيق نفسه (الكود والإعدادات):** جاهز.
- **ما ينقص:** وجود `.env`، مجلد `vendor/`، ومجلد `storage/` مع الخطوات أعلاه.
- بعد تنفيذ الخطوات وضبط `APP_URL` و `DB_DATABASE=testv02` في `.env`، يكون التحقق من بيئة التشغيل مكتملاً.
